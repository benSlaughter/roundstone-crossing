# 01 — NROD Connection & Subscription

## Account Setup

1. Register at [https://publicdatafeeds.networkrail.co.uk](https://publicdatafeeds.networkrail.co.uk)
2. After registration, log in and subscribe to the data feeds you want (TD, TRUST, etc.)
3. Credentials are your registered email and password

**Important:** Each account allows only **one concurrent STOMP connection**. If you connect a second client, the first will be disconnected silently. Use separate accounts for separate services.

## STOMP Connection

| Parameter | Value |
|-----------|-------|
| Host | `publicdatafeeds.networkrail.co.uk` |
| Port | `61618` |
| Protocol | STOMP 1.1 over TCP |
| Heartbeat | Recommended `15000,15000` (15s send/receive) |

### Python Example (using stomp.py)

```python
import stomp

conn = stomp.Connection(
    host_and_ports=[("publicdatafeeds.networkrail.co.uk", 61618)],
    heartbeats=(15000, 15000),
)
conn.set_listener("my_listener", MyListener())
conn.connect(username="your_email", passcode="your_password", wait=True)
conn.subscribe(destination="/topic/TD_ALL_SIG_AREA", id="sub-1", ack="auto")
```

### Connection Reliability

The connection can drop without warning (network issues, server maintenance, inactivity). Implement reconnection logic:

- Track connection state via `on_connected()` / `on_disconnected()` callbacks
- Use escalating backoff for reconnection attempts (e.g. 5s, 10s, 30s, 60s, 120s, 300s)
- Re-subscribe to topics after reconnecting

## Message Format

All messages arrive as **gzip-compressed byte arrays**. After decompression, the payload is a **JSON array** of message objects. Each object has a single top-level key indicating the message type:

```json
[
  {"CA_MSG": { ... }},
  {"SF_MSG": { ... }},
  {"CA_MSG": { ... }}
]
```

### Decompression (Python)

The `stomp.py` library handles decompression automatically when using `content-type: application/json`. If handling raw frames:

```python
import gzip
import json

def on_message(frame):
    body = frame.body
    if isinstance(body, bytes):
        body = gzip.decompress(body)
    messages = json.loads(body)
    for msg_wrapper in messages:
        for msg_type, msg_data in msg_wrapper.items():
            handle_message(msg_type, msg_data)
```

## Timestamps

All NROD timestamps are **Unix epoch milliseconds** (13 digits), e.g. `"1778140115000"`.

To convert: divide by 1000 to get seconds since 1970-01-01 00:00:00 UTC.

```python
from datetime import datetime, timezone
ts = int("1778140115000") / 1000
dt = datetime.fromtimestamp(ts, tz=timezone.utc)
# → 2026-05-07 07:48:35 UTC
```

**Note:** The `time` field in TD/SF messages represents when the event was generated by the signalling system, not when it was received by your client. Network and queue latency typically adds 0.5–3 seconds.
